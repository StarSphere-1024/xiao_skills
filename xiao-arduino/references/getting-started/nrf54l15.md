# XIAO nRF54L15 - Arduino Support Status

## ⚠️ IMPORTANT: Arduino NOT SUPPORTED

**The XIAO nRF54L15 does NOT support Arduino development.**

### Use MicroPython Instead

For XIAO nRF54L15 development, please use the **MicroPython** skill:
- Run `/xiao-micropython` for MicroPython support
- See `getting-started/nrf54l15.md` in the MicroPython skill

### Why Arduino is Not Supported

The nRF54L15 chipset does not have a stable Arduino core available:
- No official Arduino board package
- Community ports are experimental and incomplete
- Nordic Semiconductor's nRF Connect SDK is recommended instead

### Supported Development Platforms

| Platform | Status |
|----------|--------|
| **MicroPython** | ✅ Fully supported |
| **nRF Connect SDK** | ✅ Official Nordic SDK |
| **Zephyr RTOS** | ✅ Supported |
| **Arduino** | ❌ NOT supported |

### Board Specifications

For reference, here are the XIAO nRF54L15 specifications:

- **MCU**: nRF54L15 (Arm Cortex-M33, 64 MHz)
- **Flash**: 512KB
- **RAM**: 64KB + 64KB (retention RAM)
- **Bluetooth**: BLE 5.4 (2 Mbps, Long Range, AIS)
- **Low Power**: Ultra-low power consumption

### XIAO Standard Pin Mapping

The XIAO nRF54L15 follows the XIAO unified peripheral pin mapping:

| Interface | Pin |
|-----------|-----|
| **I2C SDA** | D4 |
| **I2C SCL** | D5 |
| **UART TX** | D6 |
| **UART RX** | D7 |
| **SPI SCK** | D8 |
| **SPI MISO** | D9 |
| **SPI MOSI** | D10 |

This ensures compatibility with XIAO expansion boards when using MicroPython.
